# Reinforcement Learning in a Custom Gridworld

Implementation of classic reinforcement learning algorithms from scratch in NumPy, applied to a custom-built gridworld environment. The goal was to develop a deep understanding of how these algorithms work at a mathematical level, without relying on RL libraries like OpenAI Gym.

---

## Environment

The gridworld is a configurable N×N grid (default 5×5) where an agent starts at a defined position and must navigate to a goal state.

- **States:** (i, j) grid coordinates
- **Actions:** `up`, `down`, `left`, `right`
- **Rewards:** -1 per step, 0 upon reaching the goal
- **Terminal state:** Goal position (default: bottom-right corner)
- **Boundary behavior:** Actions that would move the agent off the grid leave it in place (clipped)

## Algorithms Implemented

All algorithms are implemented from scratch using NumPy — no RL frameworks used.

### Monte Carlo (Every-Visit)
Estimates action-value function Q(s, a) by averaging returns from complete episodes. Uses an epsilon-soft policy derived from a uniform random policy, updated after each episode.

### Temporal Difference (TD)
Bootstraps value estimates using the Bellman equation, updating after each step rather than waiting for episode completion.

### SARSA (On-Policy TD Control)
On-policy TD control that updates Q(s, a) using the actual action taken by the current policy, following the update rule:

```
Q(s, a) ← Q(s, a) + α [r + γ Q(s', a') - Q(s, a)]
```

## Usage

```python
import gridworld

# Initialize environment (default 5x5)
grid = gridworld.gridworld()

# Custom configuration
grid = gridworld.gridworld(width=5, height=5, start=(0,0), goal=(4,4))

# Step through environment
state, reward, terminal = grid.step("right")

# Render current state
grid.render()
```

## Key Design Decisions

- **Custom environment:** Built from scratch rather than using OpenAI Gym to develop a clear understanding of the MDP formulation — states, actions, transitions, and rewards.
- **NumPy only:** All value functions and policies are represented as Python dictionaries and NumPy arrays, making the underlying math transparent.
- **Uniform random initialization:** Policy starts as a uniform distribution over actions (0.25 each), allowing algorithms to explore freely before converging.
- **Episode generation:** Episodes are generated by sampling from the current policy at each step, recording (state, action, reward) tuples for later updates.
